# 多用户生产级 Coding Agent 平台完整开发大纲

（**全链路幻觉治理深度融合版** | 基于 Spring AI Alibaba 生态 | 10 周渐进式落地 | 生产可用标准）

## 一、项目总体定位与核心目标

### 核心定位

以**Spring AI Alibaba（StateGraph + Agent Framework）** 为核心底座，基于 Supervisor Pattern 多 Agent 协作范式，构建面向 Java 开发者的、多租户隔离的代码全生命周期协作平台，实现「需求输入→代码生成→幻觉校验→安全审查→测试执行→迭代修复」的全自动化闭环，**将幻觉治理作为平台核心生命线，贯穿全流程设计**。

### 核心功能目标



1. 核心闭环能力：用户登录后创建代码任务（conversation），输入自然语言需求，触发多 Agent 系统自动完成代码全流程交付，支持长任务中断恢复、流式输出、Human-in-the-Loop 人工干预。

2. 租户级隔离：每个用户拥有独立空间，仅可管理自身的任务、代码历史、API 密钥、Agent 执行状态，数据与执行链路 100% 隔离。

3. 用户自带模型：支持用户绑定 / 临时提供多厂商 API Key（通义千问、DeepSeek、OpenAI 等），算力与 Token 成本由用户自行承担，支持多模型差异化适配。

4. **全链路幻觉治理**：构建「源头防控→Agent 交叉校验→工具层硬核验证→人工兜底→持续优化」的五层幻觉治理体系，实现代码编译通过率≥95%，基础幻觉拦截率≥99%，达到生产可用标准。

5. 安全合规：工具执行沙箱隔离、API 密钥加密存储、Prompt 注入防护、全流程审计日志，杜绝安全风险。

### 技术栈锁定



| 分层    | 技术选型                                                                                   |
| ----- | -------------------------------------------------------------------------------------- |
| 后端核心  | Spring Boot 3.3.x + Spring AI Alibaba 1.0.0 GA（StateGraph + Agent + RAG）               |
| 认证与安全 | Spring Security + JWT + 阿里云 KMS/Vault 密钥管理                                             |
| 存储层   | PostgreSQL/MySQL 8.0（主数据 + 行级安全隔离） + Redis（状态缓存 + 限流） + Spring AI VectorStore（RAG 知识库） |
| 前端与交互 | React/Vue + SSE/WebSocket（流式输出） + 代码高亮编辑器                                              |
| 工具与执行 | MCP 标准化工具 + Alibaba OpenSandbox（代码执行沙箱） + SpotBugs/PMD 静态扫描                            |
| 生产化配套 | Nacos 配置中心 + Micrometer/Prometheus 监控 + ARMS 全链路追踪                                     |

## 二、系统整体架构（幻觉治理全环节嵌入）

架构设计以「租户隔离」和「幻觉治理」为双核心，每个分层均内置对应的治理能力，而非附加功能，实现全流程无死角防控。

### 1. 表现层（Frontend）



* 核心功能：用户注册登录、个人中心、我的任务列表、任务详情页、需求输入页、人工干预审批界面

* **内置幻觉治理能力**：


  * 流式区分展示「Supervisor 决策、代码生成、幻觉校验、审查报告、测试结果」，高亮标注拦截的幻觉问题

  * 提供幻觉问题反馈入口，用户修正内容自动同步至反幻觉案例库

  * 展示当前任务 Token 消耗、幻觉拦截次数、代码编译 / 测试通过率等核心指标

### 2. 接入层（API Gateway / Controller）



* 核心功能：Spring Security JWT 认证、用户权限校验、请求路由、SSE/WebSocket 流式接口

* **内置幻觉治理能力**：


  * 统一 AOP 拦截：所有请求强制校验`userId + conversationId`归属权，杜绝上下文串扰导致的幻觉污染

  * 用户输入前置过滤：拦截 Prompt 注入、指令篡改等恶意输入，从源头避免幻觉触发

  * 输出格式前置校验：不符合 JSON Schema 规范的 Agent 输出直接拦截打回，不进入下游环节

### 3. 业务服务层（Agent Service 核心层）



* 核心功能：动态 ChatModel 工厂、Supervisor Graph 编排引擎、专精 Worker Agent 池、RAG 知识库服务

* **内置幻觉治理能力**：


  * 新增**Hallucination Checker 专属 Agent**：唯一职责为代码真实性校验，拦截编造 API、不存在依赖、语法错误等基础幻觉

  * Supervisor 节点内置幻觉拦截路由规则：所有 Worker 输出必须先通过幻觉校验，再决定下一步路由，不可绕过

  * 结构化输出强制转换：基于 Spring AI 原生`StructuredOutputConverter`，100% 约束所有 Agent 输出为预定义 JSON Schema，杜绝自由文本发散

  * 技术栈专属 RAG 知识库：注入 Spring 生态官方 API 文档、最佳实践，为代码生成提供精准知识参考，杜绝知识过时导致的幻觉

### 4. 持久化层



* 核心功能：用户信息、任务元数据、对话记忆 ChatMemory、Graph 状态快照 Checkpoint、代码文件存储

* **内置幻觉治理能力**：


  * 对话记忆持久化：基于`JdbcChatMemoryRepository`，按`conversationId`隔离，避免跨任务重复出现同类幻觉

  * 反幻觉案例库：沉淀用户修正的幻觉案例、高频幻觉场景，用于 Agent 的 Few-Shot 示例优化

  * RAG 向量库：持久化技术栈官方文档、规范标准，支持增量更新

  * 数据库行级安全 RLS：租户数据兜底隔离，避免上下文串扰导致的幻觉污染

### 5. 工具与执行层



* 核心功能：MCP 标准化工具、代码执行沙箱、Git / 文件操作、构建执行工具

* **内置幻觉治理能力**：


  * 编译校验工具：自动执行`mvn compile`，100% 拦截语法错误、不存在符号等基础幻觉

  * 依赖真实性校验工具：核对 Maven 依赖是否存在于中央仓库、版本号是否有效，杜绝编造依赖

  * 单元测试执行工具：自动运行生成的测试用例，拦截业务逻辑不符的幻觉

  * 静态代码扫描工具：集成 SpotBugs/PMD/CheckStyle，拦截安全漏洞、规范问题类幻觉

  * 沙箱隔离执行：所有代码构建、测试、运行全流程在隔离环境中执行，捕获所有执行异常，反馈给 Agent 精准修复

#### 阶段 1：核心 Agent 闭环与基础幻觉防控（2 周 | 最高优先级，先跑通核心价值）

**核心目标**：完成单用户最小可用闭环，落地成本最低、见效最快的基础幻觉防控，拦截 80% 以上的基础幻觉，验证平台核心价值。

##### 核心开发任务



1. 项目初始化与基础依赖搭建

* Spring Boot 3.3.x 项目初始化，引入 Spring AI Alibaba 核心依赖（StateGraph、Agent、ChatModel）

* 基础配置搭建，模型接入调试（通义千问 / DeepSeek 等主流代码模型）

1. 核心专精 Agent 开发与 Prompt 工程（内置反幻觉规则）

* 完成 3 个核心 Agent 开发：Planner Agent、Coder Agent、Reviewer Agent

* 为所有 Agent 编写**代码场景专属反幻觉强制 System Prompt**，锁定不可篡改的红线规则：


  * 禁止编造不存在的 API、依赖、类、方法，严格匹配 Spring Boot 3.3.x 官方规范

  * 不确定内容必须标注「待确认」，禁止无中生有，严格遵循用户原始需求

  * 每个 Agent 输出必须有明确的可验证依据，禁止模糊化、伪代码输出

* 代码场景专属模型参数调优与锁定：固定`Temperature=0.1~0.3`、`Top_P=0.2~0.4`，限制模型创造性，从采样层面减少幻觉

1. 100% 结构化输出强制约束落地

* 为所有 Agent 定义标准化 JSON Schema 输出模板，基于 Spring AI 原生`StructuredOutputConverter`实现强制转换

* 不符合格式的输出直接拦截打回，不进入下一环节，杜绝自由文本发散导致的幻觉

1. 固定流程 StateGraph 编排与闭环验证

* 基于`StateGraph`构建固定流程闭环：`Planner任务拆解→Coder代码生成→Reviewer代码审查→不通过则回Coder修复→通过则结束`

* 实现基础 SSE 流式输出，前端可实时查看 Agent 执行过程与输出内容

1. 基础幻觉校验闭环落地

* 实现 Reviewer Agent 的幻觉专项校验能力，重点核查代码的可编译性、API 真实性、依赖有效性

* 建立单环节重试机制，单 Agent 输出不通过最多重试 3 次，避免无限循环

##### 交付物



* 可本地运行的单用户代码生成 - 审查 - 修复最小闭环项目

* 3 个核心 Agent 的 Prompt 与结构化输出模板

* 基础 SSE 流式输出接口

* 核心流程测试用例与验证报告

##### 验收标准



* 简单 Spring Boot 项目需求，代码生成编译通过率≥85%

* 基础语法、编造 API 类幻觉拦截率≥80%

* 固定流程闭环可稳定复现，无卡死、无限循环问题

* 原型模式实现，每个用户的每次执行独立创建 ChatModel 实例，彻底杜绝租户间实例串扰

## 四、核心技术实现要点

### 1. 租户级用户空间隔离



* 所有操作以`(userId, conversationId)`为复合主键，全链路 AOP 权限校验，数据库行级安全 RLS 兜底隔离

* Graph 执行时强制传入`RunnableConfig`，`threadId = conversationId`，实现状态快照、对话记忆的完全隔离

* 动态 ChatModel 采用原型模式，每个用户每次执行独立创建实例，ThreadLocal 绑定请求上下文，彻底杜绝实例串扰

### 2. 用户自带 API Key 与动态模型工厂



* API 密钥采用用户级独立加密存储，基于 KMS/Vault 实现，禁止固定密钥 AES 加密，临时密钥不落库

* 基于 Spring AI 统一 ChatModel 接口，实现多厂商模型的无缝切换，上层 Agent 代码无需修改

* 前置健康检查机制，验证 Key 有效性、模型权限后再投入使用，避免执行中断

### 3. 全链路幻觉治理体系（核心融合要点）

采用五层递进式治理架构，实现从源头到兜底的全闭环防控：



1. **源头防控层**：反幻觉强制 Prompt 规则、结构化输出 100% 约束、代码场景专属模型参数锁定、用户输入注入过滤

2. **Agent 交叉校验层**：Hallucination Checker 专项校验、Supervisor 前置路由拦截、多 Agent 职责解耦交叉校验、RAG 知识库精准知识增强

3. **工具硬核验证层**：编译校验、依赖真实性校验、单元测试执行、静态代码扫描，用可执行性事实验证代码真实性

4. **人工兜底层**：高风险幻觉自动暂停、人工反馈精准注入、用户修正内容自动沉淀

5. **持续优化层**：幻觉率量化监控、反幻觉案例库沉淀、Prompt 持续迭代、分模型差异化治理

### 4. Supervisor Graph 状态机与持久化



* 基于 Spring AI Alibaba 原生`StateGraph`实现，支持动态路由、并行执行、循环修复、条件边缘判断

* 采用官方`JdbcCheckpointSaver`实现状态快照持久化，支持长任务中断恢复，无需自行实现序列化逻辑

* State 设计优化：代码文件单独存储，State 中仅保留文件路径与版本号，避免状态体积膨胀导致的性能问题

### 5. 流式输出与交互体验



* 优先基于 Spring Boot 原生 Flux 实现 SSE 流式输出，开发量低，兼容性强，后续可按需升级 WebSocket

* 流式内容按类型分类推送，区分 Supervisor 决策、代码生成、幻觉校验、审查报告，前端差异化渲染

* 支持任务执行过程中实时查看进度、暂停 / 继续执行、人工干预

### 6. 安全与沙箱执行



* 所有代码编译、测试、执行全流程在隔离沙箱中运行，单任务单容器，执行完成即销毁

* 工具执行强制超时控制、资源限制，杜绝恶意代码攻击

* 全链路审计日志，记录所有操作与执行过程，满足合规要求
